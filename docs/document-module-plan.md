# 서식 모듈 개발 구현 계획

## 개발 환경
- 프론트엔드 우선 개발 (백엔드 API는 추후 연동)
- Mock 데이터를 활용한 UI/UX 구현
- Next.js App Router 기반

## 개발 단계별 계획

### Phase 1: 기반 구조 및 서식 리스트 (1주차)

#### 1.1 서식 모듈 기본 구조 설정
- 서식 모듈 라우트 구조 설계 (`/document/form` 또는 유사 경로)
- 레이아웃 컴포넌트 구성
- Mock 데이터 구조 정의 (서식 타입, 서식 목록 등)
- 공통 타입 정의 (Form, Patient, IssueHistory 등)

#### 1.2 서식 리스트 화면 구현
- 서식 리스트 컴포넌트 (`FormList.tsx`)
- 서식 카드/아이템 컴포넌트 (`FormItem.tsx`)
- Mock 서식 데이터 생성 (법정 서식, 사용자 편집 서식)
- 페이지네이션 또는 무한 스크롤 구현

#### 1.3 서식 검색 기능
- 검색 입력 컴포넌트 (`FormSearch.tsx`)
- 검색 필터링 로직 (제목, 카테고리, 태그 등)
- 검색 결과 표시
- 검색 히스토리 (선택사항)

**완료 기준**: 사용자가 서식 리스트를 조회하고 검색할 수 있음

---

### Phase 2: 즐겨찾기 및 발급 내역 조회 (1주차 후반)

#### 2.1 즐겨찾기 기능
- 즐겨찾기 버튼 컴포넌트 (`FavoriteButton.tsx`)
- 즐겨찾기 상태 관리 (로컬 스토리지 또는 Context API)
- 즐겨찾기 필터링 기능
- 즐겨찾기 탭/섹션 추가

#### 2.2 발급 내역 조회 화면
- 발급 내역 리스트 컴포넌트 (`IssueHistoryList.tsx`)
- 발급 내역 상세 컴포넌트 (`IssueHistoryDetail.tsx`)
- 필터링 기능 (서식별, 환자별, 기간별)
- Mock 발급 내역 데이터 생성

**완료 기준**: 즐겨찾기 추가/제거 및 발급 내역 조회 가능

---

### Phase 3: 서식 발급 기본 플로우 (2주차)

#### 3.1 서식 선택 및 환자 선택 화면
- 서식 선택 모달/페이지 (`FormSelection.tsx`)
- 환자 검색 컴포넌트 (`PatientSearch.tsx`)
- 환자 선택 컴포넌트 (`PatientSelector.tsx`)
- Mock 환자 데이터 생성

#### 3.2 서식 발급 화면 기본 구조
- 서식 발급 레이아웃 (`FormIssueLayout.tsx`)
- 서식 미리보기 영역
- 환자 정보 표시 영역
- 발급 옵션 패널

#### 3.3 법정 서식 발급 기능
- 법정 서식 템플릿 컴포넌트 (`LegalFormTemplate.tsx`)
- 법정 서식 필드 렌더링
- 기본값 자동 입력 로직
- 발급 버튼 및 확인 다이얼로그

**완료 기준**: 법정 서식을 선택하여 환자에게 발급할 수 있는 기본 플로우 완성

### Phase 4: 서식 편집 및 저장 (2-3주차)

#### 4.1 서식 편집기 기본 구조
- 서식 편집기 컴포넌트 (`FormEditor.tsx`)
- 에디터 툴바 (`EditorToolbar.tsx`)

- 실시간 미리보기

#### 4.2 사용자 편집 서식 저장
- 서식 저장 기능 (`FormSaveDialog.tsx`)
- 편집 서식 목록 관리
- 서식 이름/설명 입력
- 로컬 스토리지 또는 Context로 임시 저장

#### 4.3 편집 서식 발급
- 편집 서식 선택 기능
- 편집 서식 템플릿 로드
- 환자 정보 반영 로직

**완료 기준**: 사용자가 서식을 편집하고 저장하여 발급할 수 있음

### Phase 5: 진단서 작성 및 출력 (3주차)

#### 5.1 진단서 작성 화면

## 기술 스택 및 주요 라이브러리

### 필수
- React / Next.js (App Router)
- TypeScript
- Tailwind CSS (또는 프로젝트에서 사용 중인 스타일링 솔루션)

### 추천
- **상태 관리**: Zustand 또는 Context API
- **폼 관리**: React Hook Form
- **날짜 처리**: date-fns 또는 dayjs
- **PDF 생성**: react-pdf 또는 jsPDF (출력 기능용)
- **로컬 스토리지**: localStorage 또는 IndexedDB (Mock 데이터용)

## 개발 시 주의사항

1. **컴포넌트 재사용성**: 공통 컴포넌트는 최대한 재사용 가능하게 설계
2. **타입 안정성**: TypeScript를 적극 활용하여 타입 안정성 확보
3. **에러 처리**: 각 단계에서 에러 케이스 고려
4. **로딩 상태**: 비동기 작업 시 로딩 상태 표시
5. **접근성**: 키보드 네비게이션, 스크린 리더 지원 고려
6. **반응형 디자인**: 모바일/태블릿/데스크톱 대응
7. **성능 최적화**: 불필요한 리렌더링 방지, 메모이제이션 활용
