# 개인정보 관리 프론트엔드 구현 계획 (react-frontend)

## 1. Directory Structure
`src/app/settings/personal-privacy/`

`src/components/settings/personal-privacy/`
- `DestructionTab/`:
    - `DestructionFilter.tsx`: 조회 기간, 문서 종류 필터
    - `DestructionTable.tsx`: 파기 대상 목록 테이블 (청구명세서/Receipt 기반)
    - `DestructionAlert.tsx`: 파기 확인 및 사유 입력 팝업
- `DestructionHistory/`:
    - `DestructionHistoryModal.tsx`: 파기 내역 조회 모달
    - `DestructionHistoryTable.tsx`: 내역 테이블
- `AccessHistoryTab/`:
    - `AccessHistoryFilter.tsx`
    - `AccessHistoryTable.tsx`
- `MedicalAccessHistoryTab/`:
    - `MedicalAccessHistoryFilter.tsx`
    - `MedicalAccessHistoryTable.tsx`

## 2. Page & Component Details

### 2.1 개인정보 파기 (Destruction Tab)
- **Endpoint**: `GET /personal-privacy/candidates`
- **Component**: `DestructionTab`
- **Features**:
    -   **Filter**: Date Range (Default: HospitalCreation ~ Today-5y), Document Type (Default: CLAIM_STATEMENT/Receipt).
    -   **List**: Displays `Receipt` candidates eligible for destruction.
    -   **Action**: "파기" button triggers `POST /personal-privacy/destruct`.

### 2.2 개인정보 파기 내역 (Destruction History Modal)
- **Endpoint**: `GET /personal-privacy/history`
- **Component**: `DestructionHistoryModal`
- **Features**:
    -   Opened from Destruction Tab.
    -   Displays `DestructionLog` records.

### 2.3 개인정보 접근 내역 (Access History Tab)
- **Endpoint**: `GET /personal-privacy/access-logs`
- **Component**: `AccessHistoryTab`
- **Features**:
    -   **Filter**: Date Range (Default: Last 1 month).
    -   **Table**: Timestamp, Menu Path, Action (VIEW/CREATE/UPDATE/DELETE), Patient Name.
    -   **Policy**: Visual warning or limit based on Patient Count (handled by Backend, displayed by UI if error/empty).

### 2.4 진료정보 접근 내역 (Medical Access History Tab)
- **Endpoint**: `GET /personal-privacy/medical-access-logs`
- **Component**: `MedicalAccessHistoryTab`
- **Features**:
    -   **Filter**: Date Range (Default: Last 1 month).
    -   **Table**: Timestamp, Patient Name, Chart ID (`Encounter` ID), Action, Operator.
    -   **Scope**: Logs related to Medical Chart (`Encounter`) operations.
